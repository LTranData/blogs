In the process of learning programming, you will encounter many types of data structures such as arrays, linked lists, maps, etc. Each type of data structure has its own advantages and disadvantages. Today, I will talk about an interesting type of data structure called a binary search tree, a very convenient data structure for the search problem.

1. Highlight the main problemâ

The real problems that we or businesses solve are often divided into small problems and then applying algorithms, as well as appropriate data structures to come up with a way to do it, effectively, efficiently and least costly. For the following problem, I would like to take an example from MIT's course 6.006, which you can access here.

Suppose an airline has a route management program. Each flight when it arrives at the airport must request a schedule to land at a certain time. In order not to have any conflicts, the landing times must be at least separated k{k}k minutes (1){(1)}(1). The list of landing times is R{R}R including n{n}n elements. How to add a landing time t{t}t to satisfy the constraint (1){(1)}(1) above.

I have an image to make the problem more intuitive

With the number of elements n{n}n, we want to perform a suitable position finding and insert new landing times in effective time complexity, like O(logn){O(logn)}O(logn). And the following will be the evaluations for the problem with some specific data structures.

1.1. Unsorted arrayâ

Procedure for inserting elements into an unsorted array, regardless of the constraint condition (1){(1)}(1), will cost O(1){O(1)}O(1). Procedure for inserting elements into an unsorted array, taking care of the constraint condition (1){(1)}(1), will cost O(n){O(n)}O(n).

Time complexity: O(n){O(n)}O(n).

1.2. Sorted arrayâ

  • Find the right index will costs O(logn){O(logn)}O(logn) (using binary search).
  • Compare with the element on either side of the element cost O(1){O(1)}O(1).
  • Inserting the element in the appropriate position takes O(n){O(n)}O(n) (when you will probably have to shift most of the elements up 1 position in the case of inserting the element at the beginning of the array).

Time complexity: O(n){O(n)}O(n).

1.3. Sorted linked listâ

Inserting an element into a linked list will take O(1){O(1)}O(1). But finding the inserted position will takes O(n){O(n)}O(n) when we have to traverse from head to that position.

1.4. Heap treeâ

Insertion into min-heap or max-heap

  • Finding the insertion location will takes O(n){O(n)}O(n) when we may have to traverse all elements.
  • Insertion into the min/max heap tree is unstable, because maybe after inserting in a certain position, we break the properties of the min/max heap tree and have to rerun min/max-heapify (refer to this article Fundamental Sorting Algorithms for max-heapify implementation) to get the right tree. Rerunning min/max-heapify will not able to guarantee the binding condition (1){(1)}(1).

We need a better data structure to be able to do locating and inserting element in O(logn){O(logn)}O(logn).

2. Binary Search Treeâ

Binary Search Tree is a data structure that satisfy below conditions (2){(2)}(2)

  • Each node has a maximum of 2 child nodes.
  • The value of the left child node is less than the parent node.
  • The value of the right child node must be greater than the parent node.
  • The left and right subtree is also a binary search tree.

Each node of the tree consists of

  • The value of node.
  • The pointer points to the left child node.
  • The pointer points to the right child node.
class Node:
    def __init__(self, val, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

Types of binary tree:

  • Full binary tree: each node of the tree has 0 or 2 child nodes.
  • Complete binary tree: all the tree layers are filled with nodes except for the last layer, and the bottom layer nodes must be filled from left to right
  • Degenerate binary tree: a tree where all parent nodes have only one child node.
  • Perfect binary tree: Every internal node has 2 children and the leaves are at the same level.
  • Balanced binary tree: the height of the left and right subtrees differ by at most 1.

3. Operations on binary search treeeâ

Consider a tree with n{n}n nodes, height is h{h}h.

3.1. Search - Finding a value in the treeâ

For searching for a key on the tree, we do it by recursive method. Starting at the root, we compare the value of the root node with the key. If the root node value is less than the key, we have to find it on the left subtree, if the root node value is greater than the key, we find that key on the right subtree. We do this with every node we go to. If the node value is equal to the key, we return that node. If the node value is null, we conclude that the key was not found in the tree.

Example of finding the key 40{40}40. We first compare 40<50{40 < 50}40<50, the go down the left subtree to continue to search. We compare 40>30{40 > 30}40>30, then go down the right subtree to search. Finally, we find a node whose value is 40{40}40.

Python Code

def search(root, key):
    if root is None:
        print("Cannot find the key " + key +  " in bst")
        return None
    # continue to traverse
    if root.val < key:
        return search(root.right, key)
    elif root.val > key:
        return search(root.left, key)
    else:
        return root

Algorithm analysis: finding a key in a tree will cost O(h){O(h)}O(h).

  • Average case: tree height h=Î(logn){h = \Theta(logn)}h=Î(logn), so the time complexity will be O(logn){O(logn)}O(logn).
  • Worst case: when the tree is a degenerate binary tree, tree height h=n{h = n}h=n so time complexity is O(n){O(n)}O(n).

3.2. Insert - Add a node to the treeâ

The process of inserting a node with a certain value in the tree is quite similar to search. We search until we encounter an empty node, then we insert the node we want to insert at that position. During searching, we find a node with a value equal to the key, and we return that node.

Example of inserting the key with value 4{4}4. Compare 4<6{4 < 6}4<6, go down to the left subtree, compare 4>3{4 > 3}4>3, go down to the right subtree. We encounter an empty position then we insert the key 4{4}4 at that location.

Python Code

def insert(root, key):
    # reached leaves
    if root is None:
        return Node(key)
    # continue to search, if encounter an equal value, return that node
    else:
        if root.val == key:
            return root
        elif root.val < key:
            root.right = insert(root.right, key)
        else:
            root.left = insert(root.left, key)

    return root

Algorithm analysis: finding the position to insert in the tree costs O(h){O(h)}O(h), insertion costs O(1){O(1)}O(1). Therefore, time complexity will be O(h){O(h)}O(h).

  • Average case: tree height h=Î(logn){h = \Theta(logn)}h=Î(logn), so the time complexity will be O(logn){O(logn)}O(logn).
  • Worst case: when the tree is a degenerate binary tree, tree height h=n{h = n}h=n so time complexity is O(n){O(n)}O(n).

Solving the problem in the first section: because at the step of inserting the node into the tree, we can see that we can add conditional statements to approve the insertion or not without affecting the properties in the tree at (2){(2)}(2) of a binary search tree.

As with the tree above, with constraint value k=3{k = 3}k=3. We want to insert 45{45}45. At the node 40{40}40, we have 45â40>3{45 - 40 > 3}45â40>3, so the next step is to insert 45{45}45 into the tree, without losing the properties of the tree. If we want to insert 42{42}42, we check 42â40<3{42 - 40 < 3}42â40<3, so we do not perform node insertion.

3.3. Delete - Remove a node from the treeâ

The process of deleting a node in a binary search tree occurs in 3 cases

  • The node to be deleted has no child nodes.
  • The node to be deleted has 1 child node.
  • The node to be deleted has both child nodes.

Case 1: The node to be deleted has no child nodes

Example of deleting a node with a key 40{40}40 in the tree above, we just need to release it from the tree.

Case 2: The node to be deleted has 1 child node

Example of deleting a node with a key 90{90}90 in the tree above, we just need to replace that node with its only child node.

Case 3: The node to be deleted has 2 children, we replace it with the node with the largest key in its left subtree (the rightmost node of the left subtree), or the node with the smallest key in its right subtree (the leftmost node of the right subtree).

Example of deleting a node with a key 30{30}30 in the tree above, we find the node with the smallest key in its right subtree, which is 35{35}35, and replace the node with the key 35{35}35 into the node with the key 30{30}30. The we realize that the node has a key 35{35}35 in the old position is a node with 1 child node. So we also apply the node deletion procedure like Case 2 for that location.

Python Code

# find leftmost node in the right subtree
def find_min(root):
    current = root
    while current.left is not None:
        current = current.left
    return current

def delete(root, key):
    if root is None:
        return root
    # continue to search until we find the node with the right key
    if root.val < key:
        root.right = delete(root.right, key)
    elif root.val > key:
        root.left = delete(root.left, key)
    else:
        # case 1
        if root.left is None and root.right is None:
            root = None
            return root
        # case 2
        elif root.left is None:
            root.val = root.right.val
            root.right = None
            return root
        elif root.right is None:
            root.val = root.left.val
            root.left = None
            return root
        # case 3
        else:
            temp = find_min(root.right)
            root.val = temp.val
            root.right = delete(root.right, temp.val)
            return root

    return root

Algorithmic analysis:

  • Case 1: Finding the node costs O(h){O(h)}O(h), delete the node costs O(1){O(1)}O(1). Therefore, time complexity will be O(h){O(h)}O(h).

  • Case 2: Finding the node costs O(h){O(h)}O(h), delete the node costs O(1){O(1)}O(1), node transfer costs O(1){O(1)}O(1). Therefore, time complexity will be O(h){O(h)}O(h).

  • Case 3: Finding the node costs O(h){O(h)}O(h), finding the leftmost node in the right subtree takes O(h){O(h)}O(h), deleting the leftmost node in the right subtree costs O(h){O(h)}O(h). Therefore, time complexity will be O(h){O(h)}O(h).

  • Average case: tree height h=Î(logn){h = \Theta(logn)}h=Î(logn), so the time complexity will be O(logn){O(logn)}O(logn).

  • Worst case: when the tree is a degenerate binary tree, tree height h=n{h = n}h=n so time complexity is O(n){O(n)}O(n).

3.4. Traversalâ

There are three ways to traverse and print the values ââof the nodes in the tree: pre-order, in-order, and post-order.

Consider the following tree (3){(3)}(3)

3.4.1. Pre-order traversalâ

We traverse the parent node first, to the left child node, and then to the right child node.

Example with (3){(3)}(3), in the pre-order traversal, the result will be 6,3,1,10,9,12{6, 3, 1, 10, 9, 12}6,3,1,10,9,12.

Python Code

def preorder(root):
    if root:
        print(root.val)
        preorder(root.left)
        preorder(root.right)
3.4.2. In-order traversalâ

We traverse the left child node first, to the parent node, and then to the right child node.

Example with (3){(3)}(3), in the pre-order traversal, the result will be 1,3,6,9,10,12{1, 3, 6, 9, 10, 12}1,3,6,9,10,12.

Python Code

def inorder(root):
    if root:
        inorder(root.left)
        print(root.val)
        inorder(root.right)
3.4.3. Post-order traversalâ

We traverse the left child node first, the right child node, and then the parent node.

Example with (3){(3)}(3), in the post-order traversal, the result will be 1,3,9,12,10,6{1, 3, 9, 12, 10, 6}1,3,9,12,10,6.

Python Code

def postorder(root):
    if root:
        postorder(root.left)
        postorder(root.right)
        print(root.val)

Algorithm analysis: We traverse all the nodes in the tree, so the time complexity is O(n){O(n)}O(n).
